Qt Creator Preference

发表于 2025-02-16

本文推荐了一些QT开发的配置选项供参考。

设置打开路径：

  • 在Windows或Linux平台，菜单栏选择Tools -> Options…
  • 在Mac平台，菜单栏Qt Creator -> Preferences…

构建和运行设置（Build & Run settings）

  • Save all files before build
    在编译或运行项目前保存源代码文件。
  • Compile Output: Open pane when building
    当项目构建时显示编译过程的信息。
  • Save all files before build
    在编译或运行项目前保存源代码文件。
  • Application Output: Open pane on output when running always
    在IDE底部面板显示你的程序输出。正常情况你可以看到弹出的图形控制台，无需底部面板，但是当程序窗口crashes发生，底部面板的输出可以帮助你定位问题原因。
  • Application Output: Clear old output on a new run
    清除上一次程序运行输出的信息。

调试器设置（Debugger settings）

  • Debugger font size follows main editor
    如果主编辑器字体放大，调试器字体大小将跟随放大。
  • Switch to previous mode on debugger exit
    当调试器退出后，关闭不需要的调试器窗口。

分析器设置（Analyzer settings）

Qt Creator有一个名为“Clang代码模型”的功能，它可以突出显示错误并发出代码警告。然而，这个功能有点太挑剔了，它会发出很多关于代码的警告。为了让该功能不那么挑剔，我们将更改其默认标志和设置。

在Qt Creator的首选项中，转到Analyzer区域，然后在最初的Clang Tools选项卡中，找到标记为“Default Clang Tidy and Clazy checks”的区域并单击它。现在，将弹出一个“Diagnostic Configurations”窗口，其中包含多个条目。单击“Copy…”，会弹出一个小窗口，询问您的配置名称。将您的新配置命名，然后按Enter键。在诊断配置列表中看到您的配置。确保已选中。
